# ClawHeart Desktop（桌面端）产品说明与使用指南

本文从**产品定位**到**各功能模块**、**典型操作流程**与**排障**，对 ClawHeart 桌面客户端做端到端说明。应用内 **「设置 → 文档与使用说明」**（`DocsPanel`）为精简版；需要完整结构与本说明对照时，以本文件为准。

---

## 一、产品定位与整体架构

### 1.1 是什么

ClawHeart Desktop 是运行在用户本机的 **Electron 桌面应用**，在 **`127.0.0.1:19111`** 提供：

- **管理界面（Renderer）**：总览、安全扫描、拦截监控、OpenClaw、技能市场、Agent 管理、设置与登录等。
- **本地 HTTP 服务（主进程内嵌 Express）**：UI 调用的 `/api/*`、以及面向开发者的 **OpenAI 兼容 LLM 入口**（如 `/v1/chat/completions` 等，由 LLM 代理层转发）。

策略与技能等权威数据在 **云端**（由你在设置里配置的 `apiBase`，默认如 `https://api.clawheart.live`）；本地负责 **缓存、同步、执行前校验、写本地库、转发请求**。

### 1.2 与 OpenClaw 的关系

- **完整版安装包**：内置 OpenClaw 资源，便于网关/脚本能力开箱即用。
- **Core 版**：不包含内置 OpenClaw，需自行安装或配置 OpenClaw 运行环境。
- 应用内 **「OpenClaw」** 菜单提供与本机 OpenClaw 网关相关的管理与入口（具体项以当前版本界面为准）。

### 1.3 本地数据落在哪里

本地使用 **SQLite**（`sqlite3`）持久化，数据库目录一般在用户主目录下 **`~/.clawheart`**（Windows 上为 `%USERPROFILE%\.clawheart`），文件如 **`local-client.db`**。内容包括：登录态、本地设置、技能与用户偏好、危险指令/拦截相关数据、对话历史（若开启安全扫描共享）、LLM 映射、预算与 Token 记录、Agent 目录演示数据等。

---

## 二、启动、端口与连接状态

1. 启动应用后，主进程会拉起 **本地服务（19111）**，再打开窗口加载前端。
2. 界面会请求 **`GET http://127.0.0.1:19111/api/status`** 获取状态；右下角/顶栏可反映 **与云端连接** 情况（需配置 `apiBase`、OC API Key，并已登录时更完整）。
3. **开发者本地联调**：可用 `ELECTRON_DEV=true` + Vite 开发服务器（详见项目 `package.json` 脚本），此时前端可能指向 dev server，但 API 仍为本机 19111。

## 三、界面总览

### 3.1 顶栏

- **左侧**：ClawHeart 品牌与副标题。
- **中间主导航**（可横向滚动）：**总览**、**安全扫描**、**拦截监控**、**OpenClaw**、**安全市场**（Skills）、**Agent 管理**。
- **右侧账户区**：
  - 当前用户头像首字、显示名/邮箱、连接状态点。
  - **界面语言**（自定义语言选择器，非浏览器原生 `<select>`）。
  - **明/暗主题**切换（本地 `localStorage` 记忆）。
  - **设置**（齿轮）：打开设置页（含「常规」与「文档与使用说明」）。
  - 未登录时：**登录**、**注册**。

### 3.2 主内容区

随主导航切换不同 Panel；**安全扫描** 仅在用户曾进入过该 Tab 后保持挂载（避免未使用时后台请求）。

## 四、功能模块详解

### 4.1 总览（Overview）

- **作用**：登录后聚合展示 **技能统计**、**危险指令/风险分布**、**拦截统计**、**Token 时间线**、**拦截时间线** 等（数据多来自本地代理对云端 Dashboard 接口的转发）。
- **使用**：选择时间范围、粒度等查看趋势；未登录时部分卡片无数据或提示登录。

### 4.2 安全扫描（Security Scan）

- **作用**：从云端拉取 **扫描项定义**（AI 项 + 静态项等），用户勾选后发起扫描；结果按 **严重 / 警告 / 通过** 等维度展示，并支持 **历史记录** 与详情弹窗。
- **隐私与同意**（本地存储）：
  - 是否允许使用 **共享对话历史** 参与扫描（上传前会脱敏处理）。
  - 是否 **同意读取/上传本机 AI 配置**（用于系统类扫描项）。
- **执行方式**：本地 `POST /api/security-scan/runs` 由桌面服务组包上下文（历史对话、脱敏后的本机配置等）并转发云端异步任务，前端轮询同一 run 直至完成。
- **界面语言**：扫描结果文案语言会随你在顶栏选择的 **界面语言** 传给后端（`locale`），便于与 UI 一致（具体以云端实现为准）。
- **子 Tab 典型划分**：扫描项、结果、历史（以当前版本为准）。

### 4.3 拦截监控（Intercept Monitor）

同一菜单下多个 **子 Tab**（名称以中文语言包为准）：

| 子 Tab     | 说明 |
|------------|------|
| **实时概览** | 当日 Token、费用、请求次数、平均延时等摘要。 |
| **告警记录** | 云端记录的拦截事件列表；支持类型筛选、分页、详情（含原因、片段等）。拦截类型可含：危险指令、技能禁用、预算超限等。 |
| **拦截项目** | 即参与本地匹配的危险指令规则库；支持同步、筛选、用户启用开关等（原独立「危险指令库」能力并入此处）。 |
| **请求日志** | LLM 代理路径上的请求日志；可与云端同步、查看 Provider/Model/路由/延时/费用/Token 等。 |
| **预算设置** | 按 Provider + Model 配置预算与单价；超出可触发拦截（与告警类型配合）。 |

### 4.4 OpenClaw

- **作用**：集中管理与 **OpenClaw 网关** 相关的状态与操作（例如网关启停、配置入口、内置 Node 路径等，随版本迭代可能调整）。
- **说明**：依赖本机是否安装 **完整版** 或 **Core + 自备 OpenClaw**；详见仓库 `OPENCLAW_SETUP.md`、`bundled/README.md`。

### 4.5 安全市场（Skills）

- **作用**：浏览与治理 **技能（Skills）**：官方精选/安全推荐/热门/最新等维度、系统状态（正常/禁用/不推荐）、用户侧 **启用开关**、**安全打标（SAFE/UNSAFE）**、详情与同步。
- **说明**：列表与详情会结合本地缓存与云端接口；**登录**与 **`apiBase`** 无效时常见 401 或空白。

### 4.6 Agent 管理

- **作用**：按 **Agent 平台**（如 Claude Code、Codex、Gemini CLI、OpenClaw 等）在左侧树选择 **Providers / Skills / Prompts / MCP / Sessions** 等，维护本机目录中的条目（演示/扩展数据）。
- **数据**：默认走本机 **`/api/agent-mgmt/*`**；登录后亦可与云端用户维度数据扩展同步（以当前实现为准）。

### 4.7 设置（Settings）

两个子区域：

1. **常规设置**  
   - **云端基地址**（`apiBase`）、**OC API Key**、**LLM Key**（若使用）等。  
   - **LLM 路由模式**：  
     - **通过 ClawHeart 网关（GATEWAY）**：请求经云端统一策略再转上游（推荐）。  
     - **直连（DIRECT）**：本地校验后直连你在映射/配置中的上游。  
   - **网络映射**：前缀 → 上游 Base URL（例如把 `minimax` 指到指定 API 根地址）；配合 SDK 的 Base URL 使用 `http://127.0.0.1:19111/<前缀>`。  
   - **与云端同步**技能/危险指令等（以页面按钮与说明为准）。  
   - **退出登录**（可触发本机用户态数据清理策略，以实现为准）。

2. **文档与使用说明**  
   - 内嵌 **`DocsPanel`**：快速上手、Base URL、`/v1` 配置示例、路由说明、接口速查表、FAQ 等。

### 4.8 账户（登录 / 注册）

- **登录**：向云端认证接口换取 Token，写入本地库，用于后续 `/api/*` 代理请求携带 `Authorization`。  
- **注册**：新用户开户后通常切回登录。  
- **切换账号**：顶栏账号变化后，部分页面会显示同步占位或重新拉数（避免串数据）。

## 五、开发者向：如何把 LLM 指到本地网关

1. 将 SDK 的 **Base URL** 设为：  
   **`http://127.0.0.1:19111/v1`**
2. **API Key** 可填任意占位字符串（本地网关不以此作为云端鉴权；真正上游 key 多在设置或上游配置中）。
3. 若使用 **自定义映射前缀**，则 Base URL 为 **`http://127.0.0.1:19111/<前缀>`**（无 `/v1` 后缀时取决于 SDK 如何拼接 path，需与映射设计一致）。

更短示例与说明见 **设置 → 文档与使用说明** 或下文 **接口速查**。

## 六、国际化与无障碍习惯

- **语言**：多种界面语言；安全扫描等后端文案可随所选语言返回（依赖云端与数据补丁）。

## 七、接口速查（本地 19111）

以下为常用本地 API（不完整列举，联调以代码为准）：

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/status` | 本地状态、设置摘要、登录信息 |
| GET/POST | `/api/auth/*` | 登录、注册、登出等 |
| GET/POST | `/api/settings/*` | 本地设置、路由模式、映射等 |
| GET | `/api/skills`、`POST /api/skills/sync` | 技能列表与同步 |
| PUT | `/api/user-skills/:slug`、…`/safety-label` | 用户启用与安全打标 |
| GET | `/api/intercept-logs`、`GET .../:id` | 拦截告警列表与详情 |
| GET/POST | `/api/proxy-request-logs/*` | 请求日志与云端同步 |
| GET/POST | `/api/budget/*` | 预算配置与汇总 |
| GET/POST | `/api/token-usage/*` | Token 用量与上云同步 |
| GET | `/api/security-scan/items` | 扫描项列表（转发云端，需登录） |
| POST | `/api/security-scan/runs` | 创建异步扫描任务（body 含 `itemCodes`、`context`、`locale` 等） |
| GET | `/api/security-scan/runs/:id` | 查询扫描进度与结果 |
| GET/POST | `/api/security-scan/privacy` | 读写扫描隐私同意 |
| GET | `/api/dashboard/*` | 总览图表数据代理（通常需 token） |
| GET | `/api/agent-mgmt/*` | Agent 目录本机数据 |

LLM：**`POST /v1/chat/completions`**（及同类 OpenAI 兼容路径）由 **`llm-proxy`** 按路由模式转发。

## 八、构建与分发（运维摘要）

- **Windows**：完整版 / Core 版；32 位（ia32）需本机 MSVC 编译 `sqlite3`（见 `BUILD.md`）。  
- **macOS**：需在 macOS 上执行 `build:mac` / `build:mac-no-openclaw` 等（见 `BUILD.md`、`scripts/build.js`）。  
- **内置 Node**：Windows 为 `resources/node.exe`；macOS 由 `after-pack` 写入 `.app` 的 `Resources/node`（见 `bundled/README.md`）。

## 九、常见问题（FAQ）

| 现象 | 建议 |
|------|------|
| 右下角未连接 / 总览无数据 | 检查设置里 `apiBase`、OC API Key；完成登录。 |
| 技能详情 401 | Token 过期或无效；重新登录。 |
| 安全扫描提示未登录 | 扫描转发云端需 Bearer；先登录。 |
| 策略不生效 | 看 **拦截监控 → 告警记录** 是否命中；检查 **拦截项目** 与用户启用。 |
| SDK 仍直连公网 | 确认 Base URL 已为 `http://127.0.0.1:19111/v1`（或映射前缀 URL）。 |
| 安装后闪退 | Windows 上原生模块未正确编译时可发生；按 `BUILD.md` 安装 VS Build Tools 后重新打包或联系发行方。 |
